# 局域网聊天服务

一个基于 `Node.js + React + Express + Socket.IO + SQLite` 的局域网聊天工具。

支持：

- 浏览器内创建群组、加入群组，创建时可填写房间主题名
- 通过 `roomId` 邀请同一局域网设备加入，首页列表会同时展示房间主题
- 文本聊天、图片发送、文件传输与下载（附件先上传到服务器，再由服务器中转下载）
- 服务器文件清理页：可查看服务器当前保留的聊天附件，支持全选、勾选和批量删除
- 消息撤回：群主可撤回群内任意消息，成员可在发送后 2 分钟内撤回自己的消息
- 图片在聊天框内点击即可弹窗预览，不打开新窗口
- 以设备 IP 作为唯一身份，同一 IP 使用全局固定昵称，主页可随时修改
- 群组、成员关系、聊天记录、图片元数据在服务重启后保留
- 首页自动恢复当前设备已加入/创建且仍有效的群组

## 环境要求

- `Node.js 22+`
- `npm 10+`

## 安装依赖

```bash
npm install
```

## 开发模式

同时启动后端和前端开发服务器：

```bash
npm run dev
```

- 前端开发地址：`http://localhost:5173`
- 后端 API 地址：`http://localhost:3000`

## 启动服务

```bash
npm run start -- --host 0.0.0.0 --port 3000 --data-dir ./data
```

启动后命令行会打印：

- 本机访问地址
- 当前机器可用的局域网 IPv4 地址
- 数据目录位置
- 日志目录位置

浏览器打开打印出的 URL，即可进入主页。

### 启动参数

- `--host`：监听地址，默认 `0.0.0.0`
- `--port`：监听端口，默认 `3000`
- `--data-dir`：数据目录，默认 `./data`

## 数据目录

默认数据目录结构：

```text
data/
  chat.sqlite
  uploads/
    ROOMIDXX/
      1712345678901-abcd1234.png
  logs/
    app-2026-03-09.log
    error-2026-03-09.log
```

- `chat.sqlite`：群组、成员、历史消息、昵称
- `uploads/`：图片与附件文件
- `logs/`：运行日志，按天切分，`error-*.log` 仅记录错误

## 日志

启动后日志同时输出到终端和数据目录下的日志文件：

- 普通日志：`data/logs/app-YYYY-MM-DD.log`
- 错误日志：`data/logs/error-YYYY-MM-DD.log`

当前会记录：

- 服务启动、关闭、访问地址
- HTTP 请求与响应耗时
- 昵称修改、创建 / 加入 / 退出 / 解散群组
- 文本消息、消息撤回
- 图片 / 文件上传、附件访问与下载
- Socket 连接、加入实时房间、异常错误

## 主要行为说明

- 首次创建或加入群组时，需要先填写昵称；创建群组时还需要填写房间主题
- 首页可修改全局昵称；修改后后续群组列表与在线成员昵称会同步更新
- 同一 IP 在所有群组中复用同一个昵称
- 群主创建群组后自动加入
- 群主只能解散群组，普通成员只能退出群组
- 同一 IP 在同一群组里只会保留一个活跃成员身份
- 聊天输入区域支持直接粘贴或拖拽图片 / 文件，也支持手动选择附件
- 群主可撤回任意成员消息；普通成员仅可撤回自己 2 分钟内发送的消息
- 图片支持 `png/jpg/jpeg/webp/gif` 弹窗预览；文件会以通用文件卡片展示名称与大小，并通过服务器下载
- 普通附件默认最大 `1GB`，图片仍限制 `10MB`
- 主页可进入“服务器文件清理”页面；删除后，对应聊天消息会显示该附件已被服务器清理
